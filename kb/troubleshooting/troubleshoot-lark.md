## Symptoms
- User cannot log in to Lark
- Credentials appear valid but session does not start
- User is locked out after repeated attempts
- Multi-device login conflict

## Probable Causes
- Network connectivity issues
- Incorrect credentials
- Local app cache issues
- Account restrictions from prior sessions
- Multi-device login conflict

## Resolution Steps
1. Check internet connection (WiFi or mobile data)
2. Verify username and password are correct
3. Clear Lark app cache: Settings > Apps > Lark > Storage > Clear Cache
4. Try logging in on another device to verify account status

### Password Reset
- Password resets are handled by IT department ONLY
- Do NOT attempt password resets in-app multiple times (causes lockout)
- Contact IT with your employee ID and issue description
- IT will send reset link via email within 1 hour

### Multi-Device Login
- Each user can be logged in on maximum 1 device
- To switch devices, logout from current device first
- If locked out, IT can force logout from admin panel

> **IMPORTANT:** Password reset requests must be handled by IT support only.

## Escalation Path
- Submit IT ticket with user ID, error details, and screenshot
- Request forced logout support if multi-device lock is suspected
- For urgent issues: IT Hotline available 24/7
- For non-urgent: Submit ticket through IT portal
- Provide error codes and screenshot for faster resolution

> **WARNING:** Do not perform repeated in-app reset attempts because repeated failures may trigger lockout.
